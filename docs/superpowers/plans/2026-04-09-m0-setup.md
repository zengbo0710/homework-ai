# M0 Setup & Scaffold Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Establish the monorepo foundation — npm workspaces, shared types, Fastify API stub, React PWA scaffold, MySQL via Docker, Prisma schema, and GitHub Actions CI.

**Architecture:** npm workspaces monorepo with three packages (`shared`, `api`, `web`). MySQL runs in Docker Compose; the API and frontend run on the host with hot reload. Prisma manages the full schema as a single initial migration.

**Tech Stack:** Node 20, TypeScript 5, React 18, Vite 5, Fastify 4, Prisma 5, MySQL 8, Vitest, ESLint, Prettier, GitHub Actions.

  refund

  wrong
  partial_correct

  active
  resolved

// ─── Models ───────────────────────────────────────────────────────────────────

model Parent {
  id           String   @id @default(uuid())
  email        String   @unique @db.VarChar(255)
  passwordHash String   @db.VarChar(255)
  name         String   @db.VarChar(100)
  phone        String?  @db.VarChar(20)
  locale       String   @default("en") @db.VarChar(10)

  tokenBalance      TokenBalance?
  tokenTransactions TokenTransaction[]
  children          Child[]

  @@map("parents")

  @@map("token_balances")
}

model TokenTransaction {
  id            String               @id @default(uuid())

  type          TokenTransactionType
  amount        Int
  balanceAfter  Int
  referenceId   String?              @db.VarChar(36)
  referenceType String?              @db.VarChar(30)
  description   String?              @db.Text
  paymentId     String?              @db.VarChar(255)
  createdAt     DateTime             @default(now())

  parent Parent @relation(fields: [parentId], references: [id], onDelete: Cascade)

  id        String   @id @default(uuid())
  parentId  String
  name      String   @db.VarChar(100)
  grade     Int
  avatarUrl String?  @db.VarChar(500)

  @@map("children")

  status          SubmissionStatus @default(pending)

  aiProvider      String?          @db.VarChar(30)

import { PrismaClient } from '@prisma/client';

const prisma = new PrismaClient();

async function main() {
  await prisma.systemConfig.upsert({
    where: { key: 'free_tokens_on_register' },
    update: {},
    create: {
      key: 'free_tokens_on_register',
      value: 3,
      description: 'Number of free AI tokens granted to new users',
    },
  });

  await prisma.systemConfig.upsert({
    where: { key: 'token_packages' },
    update: {},
    create: {
      key: 'token_packages',
      value: [
        { id: 'starter', tokens: 10, price_cents: 199, currency: 'USD' },
        { id: 'standard', tokens: 50, price_cents: 799, currency: 'USD' },
        { id: 'bulk', tokens: 200, price_cents: 2499, currency: 'USD' },
      ],
      description: 'Available token purchase packages',
    },
  });

  await prisma.systemConfig.upsert({
    where: { key: 'tokens_per_submission' },
    update: {},
    create: {
      key: 'tokens_per_submission',
      value: 1,
      description: 'Tokens consumed per homework scan',
    },
  });

  await prisma.systemConfig.upsert({
    where: { key: 'tokens_per_practice' },
    update: {},
    create: {
      key: 'tokens_per_practice',
      value: 1,
      description: 'Tokens consumed per practice generation',
    },
  });

  console.log('Seed complete.');
}

main()
  .catch((e) => {
