# M8/M9 — Docker & CI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Containerize the full stack (API + Web + MySQL) for prod-like local testing, then add a Docker build smoke-test to CI. Acceptance: `docker-compose -f docker-compose.prod.yml up --build` → full app at `http://localhost` in Chrome.

**Architecture:** Single infra track. Multi-stage Dockerfiles for API (Node build → slim runtime with `prisma migrate deploy` on start) and Web (Node build → nginx static). Two compose files: `docker-compose.dev.yml` (hot-reload) and `docker-compose.prod.yml` (built images + nginx). Existing `docker-compose.yml` (MySQL-only) kept for local dev without Docker API/web.

**Tech Stack:** Docker multi-stage builds, nginx:alpine, node:20-alpine, GitHub Actions.

**Spec:** `docs/superpowers/specs/2026-04-09-m8-m9-docker-ci-design.md`

- [ ] **Step 2: Create packages/api/Dockerfile**

```dockerfile
# ─── Stage 1: Build ───────────────────────────────────────────────────────────
FROM node:20-alpine AS builder
WORKDIR /app

# Copy package manifests for layer caching
COPY package*.json ./
COPY packages/api/package*.json ./packages/api/
COPY packages/shared/package*.json ./packages/shared/

RUN npm ci

# Copy source
COPY packages/shared ./packages/shared
COPY packages/api ./packages/api

# Build shared types, generate Prisma client, then build API
RUN npm run build --workspace=packages/shared
RUN npx prisma generate --schema=packages/api/prisma/schema.prisma
RUN npm run build --workspace=packages/api

# ─── Stage 2: Runtime ─────────────────────────────────────────────────────────
FROM node:20-alpine
WORKDIR /app

# Copy built output and node_modules
COPY --from=builder /app/packages/api/dist ./dist
COPY --from=builder /app/packages/api/node_modules ./node_modules
COPY --from=builder /app/node_modules/.prisma ./node_modules/.prisma
COPY --from=builder /app/node_modules/@prisma ./node_modules/@prisma

# Copy Prisma schema and migrations for migrate deploy
COPY packages/api/prisma ./prisma
COPY packages/api/package.json ./

# Ensure upload directories exist
RUN mkdir -p uploads/avatars uploads/submissions

EXPOSE 3001

# Run migrations then start server
CMD ["sh", "-c", "npx prisma migrate deploy && node dist/index.js"]
```

- [ ] **Step 3: Test the build**

```bash
docker build -f packages/api/Dockerfile . -t homework-ai-api-test
```

Expected: build succeeds, image tagged `homework-ai-api-test`.

- [ ] **Step 4: Commit**


### Task 5: nginx.conf

**Files:**
- Create: `nginx.conf`

- [ ] **Step 1: Create nginx.conf at repo root**

```nginx
events {}

http {
  include /etc/nginx/mime.types;
  default_type application/octet-stream;
  sendfile on;

  server {
    listen 80;
    server_name _;

    # API reverse proxy
    location /api/ {
      proxy_pass http://api:3001;
      proxy_set_header Host $host;
      proxy_set_header X-Real-IP $remote_addr;
      proxy_read_timeout 120s;
    }

    # Uploaded files (served by API)
    location /uploads/ {
      proxy_pass http://api:3001;
      proxy_set_header Host $host;
    }

    # Static web app (SPA fallback)
    location / {
      root /usr/share/nginx/html;
      try_files $uri $uri/ /index.html;
    }
  }
}
```

- [ ] **Step 2: Commit**

```bash

git commit -m "feat(m8): nginx reverse proxy config for API and static files"
```

```yaml
      context: .
```

- [ ] **Step 2: Create docker-compose.prod.yml**

```yaml
services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD}
      MYSQL_DATABASE: homework_ai
      MYSQL_USER: ${MYSQL_USER}
      MYSQL_PASSWORD: ${MYSQL_PASSWORD}
    volumes:
      - mysql_prod_data:/var/lib/mysql
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost", "-u", "${MYSQL_USER}", "-p${MYSQL_PASSWORD}"]
      interval: 5s
      timeout: 5s
      retries: 10

  api:
    build:
      context: .
      dockerfile: packages/api/Dockerfile
    volumes:
      - api_uploads:/app/uploads
    env_file: .env.prod
    depends_on:
      mysql:
        condition: service_healthy
    expose:
      - "3001"

  web-builder:
    build:
      context: .
      dockerfile: packages/web/Dockerfile
    volumes:
      - web_dist:/dist
    profiles:
      - build

  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
      - web_dist:/usr/share/nginx/html:ro
    depends_on:
      - api

volumes:
  mysql_prod_data:
  api_uploads:
  web_dist:
```
